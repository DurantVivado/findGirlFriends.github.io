质数1：质因数求解

设$ R = \sqrt{A}$，对于数组中每个数，最多只有一个非本身的质因数$p$满足$p\ge R$。这就意味最多只有$R+1$个不同的质因数。质因数为本身的只有一个，非本身的质因数一定比$R$小，最多$R$个。

策略

//判断是否为质数
bool judgePrime(int n) {
    for (int i = 2; i * i <= n; i++) {
        if (n % i == 0) return false;
    }
    return true;
}

质数2：质数筛选

对于一个正整数 N，一次性求出 1～N 之间所有的质数，即为质数筛选。

显然根据上述「质数判定」的内容，我们可以通过枚举 1～N 的所有数，再依次使用「试除法」来判定其是否为质数，从而完成质数的筛选。但此种方法的时间复杂度过高，为 $O(N\sqrt N)$。

我们引入了经典的[Eratosthenes]方法，也称埃氏筛。该算法基于一个基本论断：即任何质数的倍数也不是质数。

  • 我们先将2~N中的所有数标记为0；

  • 从质数2开始遍历从2~N中的所有自然数；

  • 如果遍历到一个标记为0的数，将2~N中其所有的倍数标记为1.

其代码实现比较简单，适用于绝大部分面试题：

vector<int> getPrimes(int n)
{
    vector<int> mark(n+1,0);
    vector<int> res;
    for(int i = 2;i <= n; i++)
    if(mark[i] == 0)
    {
        res.push_back(i);
        for(int j = 1;j*i <= n;j++) mark[i*j] = 1;
    }
    return res;
}

其时间复杂度为$O(N\log(\log N))$。

质数3：质因数分解

根据[唯一性分解定理]，任何一个大于1的正整数都可以分解为有限个质数的乘积：

其中，$c_i$为正整数，$p_i$为质数，且满足$p_1<p_2<\cdots<p_m$.

我们的目标是求出所有的$c_i,p_i$。

由于$p_1<p_2<\cdots<p_m$，且为$p_1$质数，因此不难发现，$p_1$ 是 N 所有因子中除 1 以外最小的数。

因此我们可以枚举$2 \to\sqrt N$中的所有数 x，如果 N 是 x 的倍数，则 x 为$p_1$。得到 $p_1$后，我们可以令 N 不断除以 $p_1$直至其不再为 $p_1$的倍数，则 $c_1$等于除以 $p_1$的总次数。

因此我们继续枚举 x，如果再次出现 N 是 x 倍数的情况，则 x 为$p_2$。

不断使用上述算法，直至循环结束。最后还需判断 N 是否为 1，如果 N 不为 1，则 $p_m=N,c_m=1$。

该算法的时间复杂度为 $O(\sqrt N)$，具体代码如下所示，大家可以配合代码对该算法进行理解：\

vector<vector<int>> primeDecomp(int N)
{
	if(N<2) return {};
	vector<vector<int>> res;
	for(int x = 2;x*x < N; x++)
	if(N%x == 0)
    {
		p = x;
		int  c = 0;
		while(N % p == 0)
		{
			c++;
			N /= p;
		}
		res .push_back({p,c});
	}
    if(N>1)
    {
        res.push_back({N,1});
    }
	return res;
}

最大公因数（Greatest Common Divisor）

交替递归，可以有效减小迭代次数。$(a,b)\rightarrow ( b,a\%b)$直到有一个数为0.

public static long gcd(long a, long b){
    if (b == 0){
        return a;
    }
    return gcd(b, a % b);
}

上述代码也可以用一行搞定：

public static long gcd(long a, long b)
{return (!b)?a:gcd(b,a%b);}

最小公倍数（LCM，Least Common Multiple）

比如 $(2,3) - 6, (3,6) - 6, (6,15) - 30$，找到$a,b$的最大公因数$k$，结果就是$a/k*b$

public static long lcm(long a, long b)
{return a/gcd(a,b)*b;}

另外，再介绍一个常见的定理 ——「裴蜀定理」：

若 a,b,x,y,m 是整数，则 ax + by = m 有解当且仅当 m 是gcd(a,b)的倍数。

该定理有一个重要的推论：整数 a,b 互质的充要条件是存在整数 x,y 使$ax+by=1$。

快速幂

参考：https://blog.csdn.net/qq_19782019/article/details/85621386

我们思考一个问题，求$2^{10E9}$的最后三位数。

  1. (a + b) % p = (a % p + b % p) % p （1）
  2. (a - b) % p = (a % p - b % p ) % p （2）
  3. (a * b) % p = (a % p * b % p) % p （3）

数学上可以证明只要我们每一步都取后三位，那么最后的结果就是答案。

快速幂的核心算法是【分治】，即把每一次幂都分为两半。

  • 如果幂为偶数则，将幂除2，将基乘2，并且取余数。
  • 如果幂为奇数，将幂减1除2，多出的一次幂乘到结果中，并且取余数。

如果最后幂为0，表示计算完毕。

我们可以适当进行优化

  • (a&1)==true表示a为奇数，因为偶数二进制最后一位一定是0.
  • 幂减1除2，可以直接省略，因为/2运算会忽略小数点。同时>>1也表示除2.
long long fastPower(long long base, long long power, int digits) {
    //digits表示取最后几位
    long long result = 1;
    while (power > 0) {
        if (power & 1) {//此处等价于if(power%2==1)
            result = result * base % digits;
        }
        power >>= 1;//此处等价于power=power/2
        base = (base * base) % digits;
    }
    return result;
}

Gauss消元法

高斯消元法主要用于求解线性方程组，也可以求矩阵的秩、矩阵的逆等，是一个重要的数学方法。其时间复杂度主要与方程组个数、方程组未知数个数有关，一般来说，时间复杂度为 $O(n^3)$

参考：https://blog.csdn.net/u011815404/article/details/88890702

高斯消元法的基本思想是：通过一系列加减消元运算，直到得到类似$Ax=b$形式，然后再解方程。

解以下几种可能性：

  1. 无解，表现为某一行系数全为0，但解不为0.
  2. 无穷多解，$det(A)=0$，增广矩阵的行列式为0，矩阵有至少一行全部为0。

思路：遍历整个上三角区域，想办法初等变换得到上三角矩阵。设增广矩阵为$matrix$，大小为$m×n$。当前行为$row$，当前列为$col$。

  • 找出当前列绝对值最大的行$maxRow$。
  • 采用冒泡法思想，将绝对值最大的行替换到最上面，方便后面进行消元。
  • 若$col$列第$row$行下全为0，则对$row—$，直接处理下一列。
  • 枚举要删去的行，采取的办法是求最小公倍数，以$maxRow$行为基准，消去$matrix[row][col],row!=maxRow$。

下面代码不保证鲁棒性，仅作理解用。

vector<int> Gauss(vector<vector<int>> &matrix)
{
/**
 * @description: 利用Gauss消元法解方程
 * @param matrix为增广矩阵，包括非齐次项
 * @return {type} None
 * @author: Durant Thorvals
 * @complexity: Time:O(1),Space:O(1)
 */    
  if(!matrix.size()||!matrix[0].size()) return {};
    int m = matrix.size(), n = matrix[0].size();
    if(m!=n-1) return {};
    vector<int> res = vector<int>(m,0);
    //转换为上三角形阶梯阵
    int row = 0, col = 0;
    for(row = 0; row < m && col < n;row++,col++)
    {
        int maxrow = row;
        //找出当前列绝对值最大的行
        for(int i = row+1;i < m;i++)
        {
            if(abs(matrix[i][col]) > abs(matrix[maxrow][col]))
                maxrow = i;
        }
        //将所有绝对值小于它的行进行交换
        if(maxrow != row)
        {
            for(int j = col;j < n;j++)
            {
                swap(matrix[row][j],matrix[maxrow][j]);
            }
        }
        
        //若col列第row行一下全为0，则处理下一列
        if(!matrix[row][col])
        {
            row -- ;
            continue;
        }
        //枚举要删去的行
        //通过寻找最小公倍数方式进行初等变换
        int lcm, t_a, t_b;
        for(int i = row+1;i < m;i++)
        {    
            lcm = LCM(matrix[row][col],matrix[i][col]);
            //采用同乘方式消元
            if(!lcm) {cout<<"The equations have infinite solution"<<endl;return {};}
            t_a = lcm/ matrix[row][col];
            t_b = lcm/ matrix[i][col];

            for(int j = col;j < n;j++)
            {//这儿就可以看出是O(n^3)时间复杂度
                matrix[i][j] = matrix[i][j]*t_b - matrix[row][j]*t_a;
            }
        }
    }
    
    //讨论解的可能形式
    //1.无解, 有一行是 0 ....0 b的形式
    int i ;
    for(i = 0;i < n;i++)
        if(matrix[m-1][i]) break; 
    if(i == n-1)
    {
        cout<<"The equations seem insolvable..."<<endl;
        return {};
    }

    //2.有无穷多解
    if(i == n)
    {
        cout<<"The equations have infinite solution"<<endl;
        return {};
    }

    //3.有唯一一组解
    //系数矩阵变为对角型，可以直接求解
    for(int i = m-1; i >= 0; i--)
    {
        int temp = matrix[i][n-1];
        for(int j = i+1;j < n-1;j++)
            temp -= res[j]*matrix[i][j];

        if(temp%matrix[i][i]!=0)
        {
            cout<<"solution needs tranform to floating unit..."<<endl;
            return {};
        }
        res[i] = temp/matrix[i][i];
    }
    return res;
    }

可以清楚的看到算法时间复杂度$O(n^3)$，设矩阵大小为$n$。

中位数

295. 数据流的中位数(Hard)

480. 滑动窗口中位数(Hard)

如果数列长度为奇数，中位数是一个排序数列最中间的元素；否则中位数是最中间两个数的平均值，所以求中位数 的函数必定为double型。

一种常见的做法是采用排序的方法，但这样做最优的时间复杂度为$O(NlogN)$，我们需要更好的方法。

这儿有两种做法：1. 堆， 2. 二叉平衡搜索树(BST)

先来看看堆的做法，我们试着维护两个有序数组，存储较大一半数字的大顶堆和存储较小一半数字的小顶堆。

  • 前有序数组由于只关注最大值，可以 动态地 放置在一个最大堆中；

  • 后有序数组由于只关注最小值，可以 动态地 放置在一个最小堆中。

  • 当从数据流中读出的数的个数为偶数的时候，我们想办法让两个堆中的元素个数相等，两个堆顶元素的平均值就是所求的中位数（如下左图）；

  • 当从数据流中读出的数的个数为奇数的时候，我们想办法让最大堆的元素个数永远比最小堆的元素个数多 1 个，那么最大堆的堆顶元素就是所求的中位数（如下右图）。

具体可以进行如下操作：

情况 1： 当两个堆的元素个数之和为偶数（例如一开始的时候），为了让最大堆中多 1 个元素，采用这样的流程：「最大堆 → 最小堆 → 最大堆」；

情况 2： 当两个堆的元素个数之和为奇数，此时最小堆必须多 1 个元素，这样最大堆和最小堆的元素个数才相等，采用这样的流程：「最大堆 → 最小堆」 即可。

因此无论两个堆的元素个数之和是奇数或者是偶数，都得先「最大堆」 再「最小堆」 ，而当加入一个元素之后，元素个数为奇数的时候，再把最小堆的堆顶元素拿给最大堆就可以了。

将元素放入优先队列以后，优先队列会自行调整（以对数时间复杂度），把最优值放入堆顶，是这道问题思路的关键。

技巧：对偶数和奇数长度的数列，我们可以用(nums[mid]+nums[mid-(1-k%2)])/2表示中位数。

另外480题要求优先队列的「延迟删除」技巧，笔者也专门学习了一下：

当我们需要移出优先队列中的某个元素时，我们只将这个删除操作「记录」下来，而不去真的删除这个元素。当这个元素出现在小顶堆或者大顶堆的堆顶时，我们再去将其移出对应的优先队列。

为此我们需要一个辅助的数据结构——哈希表，保存的键值对$(num,freq)$表示元素num需要被删除freq次。

279. 完全平方数

给定正整数 n，找到若干个完全平方数（比如 1, 4, 9, 16, ...）使得它们的和等于 n。你需要让组成和的完全平方数的个数最少。

示例 1:

输入: n = 12
输出: 3 
解释: 12 = 4 + 4 + 4.

示例 2:

输入: n = 13
输出: 2
解释: 13 = 4 + 9.

1770年，Lagrange)证明了一个定理，被称为四平方数定理，它指出每个自然数都能表示为四个平方整数之和：

这就规定了问题的上限，即最大的平方数个数是4.

注意，这里0也被当成完全平方数。

后来1797年，Legendre用他的三平方定理完成了四平方定理，正整数可以表示为三个平方和的一个特殊条件：

在我们分解为（底部情况）三平方数之前，还有两种特例需要检查：

  1. 该数本身是一个完全平方数 即n=(int)(sqrt(n)*sqrt(n))

  2. 如果一个数可以分解为两个平方数之和，不幸的是，我们只能通过枚举来找到这样的数。

算法：

  1. 检查该数是否为完全平方数

  2. 如果该数可以表示为$n == 4^k(8m+7)$，那么可以确切的说，答案肯定为4；

while(n%4==0) n /= 4;
if((n-7)%8==0) res = 4;
  1. 我们看该数，是否能由更少的完全平方数组成。
for(int k = n-1;k >= 1;k--)
{
    if(isPerfectSquare(n*n-k*k)) res = 2;
}

代码：

class Solution {
public:
    bool isPerfectSquare(int n)
    {
        return n == ((int)sqrt(n))*((int)sqrt(n));
    }
    int numSquares(int n) {
    //一些很特殊的情况12 = 2^2+2^2+2^2; 18 = 3^2+3^2;1000= 30^2+10^2;61=5^2+6^2
    if(isPerfectSquare(n)) return 1;
    int nn = n;
    //判断是否为四平方数
    while(nn%4==0) nn/=4;
    if((nn-7)%8==0) return 4;
    for(int k = n-1;k >= 1;k--)
    if(isPerfectSquare(k))
    {
        // printf("%d:\n",k);
        if(isPerfectSquare(n-k)) return 2;
    }
    return 3;

    }
};

10. 可见点的最大数目

给你一个点数组 points 和一个表示角度的整数 angle ，你的位置是 location ，其中 location = [posx, posy] 且 points[i] = [xi, yi] 都表示 X-Y 平面上的整数坐标。

最开始，你面向东方进行观测。你 不能 进行移动改变位置，但可以通过 自转 调整观测角度。换句话说，posx 和 posy 不能改变。你的视野范围的角度用 angle 表示， 这决定了你观测任意方向时可以多宽。设 d 为你逆时针自转旋转的度数，那么你的视野就是角度范围 [d - angle/2, d + angle/2] 所指示的那片区域。

同一个坐标上可以有多个点。你所在的位置也可能存在一些点，但不管你的怎么旋转，总是可以看到这些点。同时，点不会阻碍你看到其他点。

返回你能看到的点的最大数目。

示例 1：

输入：points = [[2,1],[2,2],[3,3]], angle = 90, location = [1,1]
输出：3
解释：阴影区域代表你的视野。在你的视野中，所有的点都清晰可见，尽管 [2,2] 和 [3,3]在同一条直线上，你仍然可以看到 [3,3] 。

示例 2：

输入：points = [[2,1],[2,2],[3,4],[1,1]], angle = 90, location = [1,1]
输出：4
解释：在你的视野中，所有的点都清晰可见，包括你所在位置的那个点。

示例 3：

输入：points = [[1,0],[2,1]], angle = 13, location = [1,1]
输出：1
解释：如图所示，你只能看到两点之一。

提示：

  • 1 <= points.length <= 105
  • points[i].length == 2
  • location.length == 2
  • 0 <= angle < 360
  • 0 <= posx, posy, xi, yi <= 109
