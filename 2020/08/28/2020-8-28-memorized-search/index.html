记忆化搜索vs动态规划

记忆化搜索vs动态规划

  • 记忆化搜索实际上是递归来实现的，但是递归的过程中有许多的结果是被反复计算的，这样会大大降低算法的执行效率。　而记忆化搜索是在递归的过程中，将已经计算出来的结果保存起来，当之后的计算用到的时候直接取出结果，避免重复运算，因此极大的提高了算法的效率。

记忆化搜索与动态规划区别：记忆化搜索：「自顶向下」，动态规划：「自底向上」。所有动态规划能解决的问题，记忆化搜索也能解决，唯一要注意的是动态规划不容易超时，但空间占用大，而记忆化搜索因为递归较多，容易TLE。

312. 戳气球

有 n 个气球，编号为0 到 n-1，每个气球上都标有一个数字，这些数字存在数组 nums 中。

现在要求你戳破所有的气球。如果你戳破气球 i，就可以获得 nums[left] * nums[i] * nums[right] 个硬币。 这里的 left 和 right 代表和 i 相邻的两个气球的序号。注意当你戳破了气球 i 后，气球 left 和气球 right 就变成了相邻的气球。

求所能获得硬币的最大数量。

说明:

你可以假设 nums[-1] = nums[n] = 1，但注意它们不是真实存在的所以并不能被戳破。
0 ≤ n ≤ 500, 0 ≤ nums[i] ≤ 100
示例:

输入: [3,1,5,8]
输出: 167 
解释: nums = [3,1,5,8] --> [3,5,8] -->   [3,8]   -->  [8]  --> []
     coins =  3*1*5      +  3*5*8    +  1*3*8      + 1*8*1   = 167

总的戳破气球操作数一定为n。我们可以倒过来看这些操作，每一次添加一个气球。

我们定义方法$solve(i,j)$表示将区间$(i,j)$全部填满所能得到的最多的硬币数。这里是一个开区间。当

  • $i\ge j-1$，$solve(i,j)=0$

  • 否则，我们枚举开区间$(i,j)$内的全部位置$mid$，令$mid$为当前区间第一个添加的气球，该操作所能得到的硬币数为

  • 同时我们递归的计算出分割的两区间对$solve(i,j)$的贡献，这三项最大值，即为$solve(i,j)$的值。这样问题可以转换为求$solve(i,mid)$与$solve(mid,j)$之和。

状态转移方程：

然后我们再看边界条件：

$solve(0,0)=0$

为了避免重复计算，我们存储solve的中间结果。

class Solution {
public:
    int n; 
    vector<vector<int>> memo;
    vector<int> nums;
    int solve(int i, int j)
    {//开区间(i,j)内积分最大值,先加mid，再计算最大值
        
        if(i >= j-1) return 0;
        
        if(memo[i][j]!=-1) return memo[i][j];
        for(int mid = i+1; mid < j; mid++)
        {
            memo[i][j] = max(memo[i][j], nums[i]*nums[mid]*nums[j]+solve(i,mid)+solve(mid,j) );
        }
        return memo[i][j];
    }
    int maxCoins(vector<int>& nums) {
        if(!nums.size()) return 0;
        //这一题和移除盒子是同类型题，要靠JOJO奇妙的动态规划来解决
        //我们要让尽可能大的气球相邻
        n = nums.size();
        nums.insert(nums.begin(),1);
        nums.emplace_back(1);
        this->nums = nums;

        memo.resize(n+2,vector<int>(n+2,-1));
        return solve(0,n+1);
    }
};

复杂度分析

  • 时间复杂度：$O(n^3)$，其中 $n$ 是气球数量。区间数为 $n^2$，区间迭代复杂度为 O(n)O(n)，最终复杂度为 $O(n^2 \times n) = O(n^3)$

  • 空间复杂度：$O(n^2)$，其中 $n$ 是气球数量。缓存大小为区间的个数。

动态规划代码


class Solution {
public:
    int maxCoins(vector<int>& nums) {
        int n = nums.size();
        vector<vector<int>> rec(n + 2, vector<int>(n + 2));
        vector<int> val(n + 2);
        val[0] = val[n + 1] = 1;
        for (int i = 1; i <= n; i++) {
            val[i] = nums[i - 1];
        }
        for (int i = n - 1; i >= 0; i--) {
            for (int j = i + 2; j <= n + 1; j++) {
                for (int k = i + 1; k < j; k++) {
                    int sum = val[i] * val[k] * val[j];
                    sum += rec[i][k] + rec[k][j];
                    rec[i][j] = max(rec[i][j], sum);
                }
            }
        }
        return rec[0][n + 1];
    }
};

546. 移除盒子

给出一些不同颜色的盒子，盒子的颜色由数字表示，即不同的数字表示不同的颜色。
你将经过若干轮操作去去掉盒子，直到所有的盒子都去掉为止。每一轮你可以移除具有相同颜色的连续 k 个盒子（k >= 1），这样一轮之后你将得到 k*k 个积分。
当你将所有盒子都去掉之后，求你能获得的最大积分和。

示例：

输入：boxes = [1,3,2,2,2,3,4,3,1]
输出：23
解释：
[1, 3, 2, 2, 2, 3, 4, 3, 1] 
----> [1, 3, 3, 4, 3, 1] (3*3=9 分) 
----> [1, 3, 3, 3, 1] (1*1=1 分) 
----> [1, 1] (3*3=9 分) 
----> [] (2*2=4 分)

提示：

1 <= boxes.length <= 100
1 <= boxes[i] <= 100

这一题的规模比较小，按照官方的尿性，官解的时间复杂度会很大。很神奇的是，这种题目也可以通过动态规划解决。

用$f(l,r,k)$表示移除区间$[l,r]$加上该区间右边等于$a_r$的$k$个元素组成的序列的最大积分。

486. 预测赢家

给定一个表示分数的非负整数数组。 玩家 1 从数组任意一端拿取一个分数，随后玩家 2 继续从剩余数组任意一端拿取分数，然后玩家 1 拿，…… 。每次一个玩家只能拿取一个分数，分数被拿取之后不再可取。直到没有剩余分数可取时游戏结束。最终获得分数总和最多的玩家获胜。

给定一个表示分数的数组，预测玩家1是否会成为赢家。你可以假设每个玩家的玩法都会使他的分数最大化。

示例 1：

输入：[1, 5, 2]
输出：False
解释：一开始，玩家1可以从1和2中进行选择。
如果他选择 2（或者 1 ），那么玩家 2 可以从 1（或者 2 ）和 5 中进行选择。如果玩家 2 选择了 5 ，那么玩家 1 则只剩下 1（或者 2 ）可选。
所以，玩家 1 的最终分数为 1 + 2 = 3，而玩家 2 为 5 。
因此，玩家 1 永远不会成为赢家，返回 False 。

示例 2：

输入：[1, 5, 233, 7]
输出：True
解释：玩家 1 一开始选择 1 。然后玩家 2 必须从 5 和 7 中进行选择。无论玩家 2 选择了哪个，玩家 1 都可以选择 233 。
     最终，玩家 1（234 分）比玩家 2（12 分）获得更多的分数，所以返回 True，表示玩家 1 可以成为赢家。

提示：

  • 1 <= 给定的数组长度 <= 20.
  • 数组里所有分数都为非负数且不会大于 10000000 。
  • 如果最终两个玩家的分数相等，那么玩家 1 仍为赢家。

我们用$dp[l][r]$表示闭区间$[l,r]$内，自己「先手」能获得的分数最大值。那么函数dfs(l,r)返回dp[l][r]。并进行记录。

记忆化搜索

$dp[l][r] = sum(l,r) - min(dp[l+1][r],dp[l][r-1])$

边界条件 $dp[l][l] = nums[l]$; 返回值：$dfs(nums,0,n-1)*2 >= sum(nums,0,n-1)$

时间复杂度：$O(2^n)$

空间复杂度: $O(n)$，递归的空间复杂度取决于使用了多少栈空间。

动态规划

$dp[l][r] = max(-dp[l+1][r] + nums[l],-dp[l][r-1]+nums[r])$

我们先在$nums[l],nums[r]$中进行选择，同时保证对手的$dp[l+1][r]或者dp[l][r-1]$最小。

边界条件 $dp[l][l] = nums[l]$, 返回值：$dp[0][n-1]>=0$

使用递归存在大量重复计算，时间复杂度很高，因而采用动态规划是更好的选择。注意：$i$的遍历要从$n$开始，即反过来。

时间复杂度：$O(n^2)$

空间复杂度: $O(n)$，使用滚动数组优化。

代码（DFS+记忆化搜索）

class Solution {
public:
    int dp[22][22];
    int allsum(vector<int>& nums,int l, int r)
    {//对下标l-r进行求和
        int sum = 0;
        for(int i = l; i <= r;i++) sum += nums[i];
        return sum;
    }
    int dfs(vector<int> &nums, int l,int r)
    {//记忆化搜索，返回l,r使得先手优势最大的值

        if(dp[l][r]!=-1) return dp[l][r];
        if(l>r) return 0;
        if(l==r) {dp[l][r]=nums[l];return nums[l];}
        dp[l][r] = allsum(nums,l,r) - min(dfs(nums,l+1,r),dfs(nums,l,r-1));
        // printf("dp[%d][%d]=%d\n",lr,dp[l][r]);
        return dp[l][r];
    }
    bool PredictTheWinner(vector<int>& nums) {
    //采用零和博弈的动态规划解法
    int n = nums.size();
    memset(dp, -1, sizeof(dp));
    return dfs(nums,0,n-1)*2>=allsum(nums,0,n-1);
    }
};

代码（DFS+动态规划）

class Solution {
public:
    
    bool PredictTheWinner(vector<int>& nums) {
    //采用零和博弈的动态规划解法
    int n = nums.size();
    vector<vector<int>> dp(n,vector<int>(n,-1));//表示[i,j]区间的最大值
    for(int i = 0;i < n;i++) dp[i][i] = nums[i];
    
    for(int i = n;i >= 0;i--)
    for(int j = i+1;j < n;j++)
    if(i+1<n && j-1>=0)
    {
        dp[i][j] = max(-dp[i+1][j]+nums[i], -dp[i][j-1]+nums[j]);
        // cout<<i<<","<<j<<","<<dp[i][j]<<endl;    
        
    }
    
    return dp[0][n-1]>=0;
    }
};

464. 我能赢吗

在 “100 game” 这个游戏中，两名玩家轮流选择从 1 到 10 的任意整数，累计整数和，先使得累计整数和达到或超过 100 的玩家，即为胜者。

如果我们将游戏规则改为 “玩家不能重复使用整数” 呢？

例如，两个玩家可以轮流从公共整数池中抽取从 1 到 15 的整数（不放回），直到累计整数和 >= 100。

给定一个整数 maxChoosableInteger （整数池中可选择的最大数）和另一个整数 desiredTotal（累计和），判断先出手的玩家是否能稳赢（假设两位玩家游戏时都表现最佳）？

你可以假设 maxChoosableInteger 不会大于 20， desiredTotal 不会大于 300。

示例：

输入：
maxChoosableInteger = 10
desiredTotal = 11

输出：
false

解释：
无论第一个玩家选择哪个整数，他都会失败。
第一个玩家可以选择从 1 到 10 的整数。
如果第一个玩家选择 1，那么第二个玩家只能选择从 2 到 10 的整数。
第二个玩家可以通过选择整数 10（那么累积和为 11 >= desiredTotal），从而取得胜利.
同样地，第一个玩家选择任意其他整数，第二个玩家都会赢。

首先我们弄清稳赢的含义，也就是说在某一个回合，A走完一步后，B发现他无论下哪一步都会输，或者A发现自己无论下哪一步都会赢。这两者都表示A会赢。

这道题目，是和博弈论相关的「先手」，很容易想到的做法是动态规划，但是因为回合数不确定，很难直接写出状态转移方程，这个时候我们可以，另辟蹊径，采用「记忆化搜索」，定义一个helper函数，参数分别为记忆化搜索范围int maxChoosableIntegerx，目标int target,状态转移方程组boolean[] dp, 还有状态int state，框架在两分钟内就可以搭好。

state|(1<<(i-1));//表示对state第i位置1
state & 1<<(i-1);//判断state第i位是否为1

我们思考状态压缩，因为最大整数不超过20，这就提示我们可以采用「状态压缩」。用$N$位二进制来表示状态，1表示选择，0表示未选，例如1000_010表示选择2和7。这里dp作用是记忆状态，如果在递归时，我们发现dp[state]!=null就可以直接返回dp[state]。

算法

我们实际考虑每个回合，先手能否稳赢，这里先手指各个回合。设i为选择的数，如果这个回合target - i <= 0或者下一个回合 helper(N, target - i, dp, state|cur) == false 则表明先手获胜dp[state] = true并返回。

代码


class Solution {
    Boolean[] b;

    public boolean canIWin(int maxChoosableInteger, int desiredTotal) {
        //这个题属于经典零和博弈问题，要判断先手的玩家能否赢
        //如果最大整数小于等于目标整数，那么先手玩家一定会赢
        //记忆化搜索+状态压缩
        if(maxChoosableInteger >= desiredTotal) return true;
        if((1+maxChoosableInteger)*maxChoosableInteger < 2*desiredTotal) return false;
        b = new Boolean[1 << maxChoosableInteger];
        return helper(desiredTotal, 0, b , maxChoosableInteger);
    }
    public boolean helper(int target, int state, Boolean[] dp,int N)
    {//dp[i]表示选择第i位后的状态
        if(dp[state] != null)
            return dp[state];//用类类型代替封装
        for(int i = N;i >= 1;i--)
        {
            if((state & 1<<(i-1)) == 0) //说明第i位还没用过
            {
                int cur = 1<<(i-1);//将第i位置1

                if( target <= i || (helper(target - i, state|cur, dp, N) == false)) {
                    return dp[state] = true;
                }
            }

        }
        return dp[state] = false;

    }



}

复杂度分析

时间复杂度：$O(N!)$

空间复杂度：$ O(1<<m)$，$m$为最大可选择整数。
